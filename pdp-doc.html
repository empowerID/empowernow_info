What Makes AuthZen PDP Different

Five breakthrough capabilities that set us apart

1

Delegation Graph

User-bound AI agents with CONTROLLED_BY edges and spend caps on DELEGATES_TO relationships. Full graph traversal for complex authorization chains.

Batch AuthZen Evaluation

3

Learning Mode

Test new policies in production without enforcement. Compare decisions against existing system, roll out with confidence.

4

Explainability

Every decision comes with deterministic reasons and auto-generated Mermaid diagrams showing the evaluation path.

5

Constraints vs Obligations

Pre-conditions (constraints) that PEP must enforce vs post-actions (obligations) like audit logs, notifications, or approvals.

Real Policy Examples

Production-ready YAML from our sample library 

# User delegates to AI agent with financial constraints
id: ai-agent-delegation-policy
version: "1.0"
subjects:
  - type: ai_agent
    condition: |
      principal.type == "SERVICE" AND
      principal.controlled_by == ctx.attributes.user_id
      # CONTROLLED_BY edge resolved by PIP

resources:
  - type: booking
    match: resource.type == "flight"

actions:
  - purchase

effect: allow

constraints:
  - id: spend_limit
    type: numeric_cap
    parameters:
      amount: 4500
      currency: USD
  - id: velocity_check
    type: txn_velocity
    parameters:
      window_minutes: 1440
      max_count: 5

obligations:
  - id: notify_delegator
    type: workflow_exec
    parameters:
      workflow_id: "notify_email"
      target: ctx.attributes.user_email
  - id: audit_trail
    type: log_activity
    parameters:
      category: user_activity
      retention_days: 30
