Seven Controls — Plain Language

1) User‑Bound Agent Identity

Each agent instance is cryptographically tied to one user—eliminates cross‑user data leaks and clarifies accountability.

2) Tool Schema Attestation

Only call approved tool versions; schema hashes must match or calls fail closed.

3) Capability Proofs

Prove the one permission you’re using (e.g., refund.issue) without revealing your full capability set.

4) Plan Contracts

Pre‑signed steps with parameter fingerprints and hard budget caps prevent “off‑script” behavior.

5) Context‑Root Binding

Fingerprint trusted inputs (prompts, policy text, prior outputs); if they change, execution stops.

6) Behavioral DNA

Behavioral baselines detect agent drift and compromise; enforce, alert, or require re‑auth.

7) Receipt Chains

Signed, hash‑linked receipts create a tamper‑evident audit for every allow/deny.
