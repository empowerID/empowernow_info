What is OpenID Connect (OIDC)?

OIDC is an identity layer on top of OAuth 2.0. It uses ID tokens (JWT) to convey who the user is and how they authenticated.

Why it matters

  • Standard way to sign users in to applications
  • Interoperable identity across providers and platforms
  • Extensible with profiles like FAPI 2.0 for high assurance

How it works (at a glance)

  1. Client requests authentication using OIDC scopes (e.g., openid, email)
  2. Authorization Server returns an ID token (JWT) and access token
  3. Client verifies the ID token and optionally calls the UserInfo endpoint

Key terms

  • ID Token: JWT with user identity claims
  • UserInfo: Endpoint to fetch user attributes
  • Claims: Pieces of identity information
