What is PKCE?

Proof Key for Code Exchange binds the authorization code to the client instance, preventing interception attacks in public clients like SPAs and mobile apps.

Why it matters

  • Mitigates code interception and replay
  • Recommended for all OAuth code flows (public and confidential)

How it works (at a glance)

  1. Client generates a code verifier and derives a code challenge
  2. Sends the challenge in the authorize request
  3. Exchanges the code with the original verifier; AS verifies the binding

Common pitfalls

  • Storing tokens in the browser even with PKCE. Prefer BFF with cookies.
  • Using plain (S256 is recommended)
