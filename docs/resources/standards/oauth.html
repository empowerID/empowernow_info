What is OAuth 2.0?

OAuth 2.0 is an authorization framework that lets a client access an API on behalf of a resource owner using time‑bounded access tokens instead of passwords.

Why it matters

  • Removes the need to share credentials with applications
  • Scopes constrain what a token can do
  • Standardized flows enable secure, interoperable integrations

How it works (at a glance)

  1. The client sends the user to the Authorization Server to request consent
  2. On consent, the client obtains an authorization code and exchanges it for tokens
  3. The client calls the API with the access token; the API validates and serves data

Key terms

  • Authorization Server (AS): Issues tokens after consent
  • Resource Server (RS): API that validates tokens
  • Client: App requesting access on the user’s behalf
  • Scopes: Permissions attached to tokens

Common pitfalls

  • Storing tokens in the browser (risk of theft). Prefer BFF with HTTP‑only cookies.
  • Overly broad scopes. Prefer least‑privilege via RAR and resource‑level policy.
  • Long‑lived tokens. Prefer short lifetimes and refresh rotation.
