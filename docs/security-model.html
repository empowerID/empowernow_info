EmpowerNow Security Model

From login to audit trail in <150 ms. This document explains how Authentication Studio, Authorization Studio and Automation Studio weave identity, policy and observability into every request.

1. End-to-end sequence

The diagram shows a user triggering a workflow from Automation Studio. Identity flows from Authentication Studio to Authorization Studio before any connector fires.

2. DPoP proof

Each outgoing request is signed by the BFF using the session's private key. Workflow nodes reject mismatched proofs to prevent replay.

3. PDP decision

Authorization Studio evaluates AuthZEN policies (<20 ms p95) using contextual attributes supplied by PIPs (GeoIP, membership graph, risk score).

4. CAEP events

Every mutation emits a Cloud Audit pattern event containing subject, object, action, PDP decision, and correlation IDs. Events are streamed to your SIEM or data lake.

5. OTEL correlation

All components emit OpenTelemetry spans with the X-Correlation-ID. This lets you click from a user action in the UI to the exact SQL statement executed by a connector.
