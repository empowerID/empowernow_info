Tool Registry

Catalog of MCP/HTTP tools enabling verifiable request shaping via hash‑pinned schemas with safe rollouts.

CURRENT/pin semantics

  • Exactly one CURRENT version per tool
  • Pins include { schema_version, schema_hash } from canonical JSON
  • Rollout grace windows allow accepting the previous version temporarily
  • ETag on GET enables conditional requests

Read endpoints (hot path)

GET  /tools/{tool_id}
HEAD /tools/{tool_id}
GET  /tools/{tool_id}/pin   (?format=jws)

GET returns { id, endpoint, schema_version, schema_hash, previous_version, previous_hash, updated_at, grace_seconds, risk_tier, auth_mode, cost_per_call } with ETag and Cache-Control.

Admin flows (write path)

  • Create tool, add versions, and flip CURRENT atomically
  • Optional signed pin JWS for integrity

Why it matters

Prevents schema drift and enables safe version flips without breaking agents. Caches efficiently for high QPS.
