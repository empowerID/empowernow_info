Where ARIA Fits

How ARIA differs from AI gateways, authorization engines, and workflow tools.

ARIA vs. “AI Gateways”

  • Focus: Gateways emphasize observability and cost estimates; ARIA enforces AuthZEN decisions at runtime with plan/budget gating and receipts.
  • Identity: Gateways rarely carry user‑bound agent identity; ARIA uses Passports with pairwise IDs and optional DPoP.
  • Control: Gateways can’t enforce per‑agent plan/budget pre‑execution; ARIA denies before tool calls and returns clear 402 on exceed.

ARIA vs. Authorization Engines

  • Scope: PDPs decide; ARIA decides + enforces (MCP PEP) with schema/params/egress, plan/budgets, receipts.
  • Context: ARIA integrates Membership PIP for data_scope, step_up, identity_chain.

ARIA vs. Workflow Tools

  • Model: Generic tools orchestrate UIs/tasks; ARIA Orchestration is AI‑native (zero‑shot, next paths, node decisions, Mermaid).
  • Governance: ARIA couples identity + policy + enforcement + receipts end‑to‑end.
