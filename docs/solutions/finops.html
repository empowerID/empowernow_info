FinOps Governance

Predictable spend with runtime guardrails and auditable receipts.

The problem

AI usage sprawls; traditional gateways show cost, but rarely enforce budgets or plans in real time.

The ARIA approach

  • Per‑agent budgets (idempotent debits keyed by call_id)
  • Deny before tool execution when PDP says no
  • Plan contracts (JWS) and schema pins govern execution
  • Signed, hash‑chained receipts for reconciliation

Outcomes

  • Predictable spend
  • Auditable usage trails
  • Safer experimentation

Pilot outline (2–3 weeks)

  • Week 1: connect Registry; model budget policy; enable receipts
  • Week 2: enforce plan/budgets in Gateway/BFF; dashboards
  • Week 3: expand; document runbook; tune PDP TTLs
